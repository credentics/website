Most identity infrastructure was designed for a world of long-lived servers and
long-lived secrets. Cloud-native systems don't work that way — workloads come
and go in seconds, and the credentials that protect them shouldn't outlive the
process that requested them.

That's the problem we started Credentics to work on: **identity and security
tooling built for cloud-native platforms from the ground up**, with OIDC and
short-lived, verifiable credentials as the default instead of an afterthought.

## Why open source

Security software that you can't read is security software you have to take
on faith. We don't think that's good enough for infrastructure this
foundational, so every project we ship will be open source under a permissive
license from its first commit. If we ask you to trust a system with your
workload identity, you should be able to go read exactly what it does.

## Where we're starting

We're a brand-new studio. There's no public tool to try yet —
we're still heads-down building the first release. Over the next few months
we'll be sharing progress here on the blog: design decisions, early previews,
and the tradeoffs we ran into along the way.
